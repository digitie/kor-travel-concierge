# UI 레벨 수집 E2E 리포트 (10영상×3소스) — 2026-06-20

## 실행 환경

  최대 영상 수 10 → "수집 시작" → "자막 생성 시작"(키워드는 UI 버튼, 채널/플레이리스트는 동일 엔드포인트).
- **코드**: 최신 `main`(DeepSeek 엔진·사전 프롬프트·#91 설명→POI) + **T-089 타임스탬프 버그 수정**을
  반영해 `kor-travel-docker-manager` concierge를 재빌드(같은 12601/12605 포트).
- **DB**: 깨끗한 `kor_travel_concierge_e2e`(빈 상태, 이 실행 데이터만). 엔진 Gemini 2.5 Flash, 지오코딩 VWorld.
- **입력**: 채널 `@빵이네tv`(`UCnV8h6ZzQnLoFBFXqHGtxBg`), 플레이리스트 `PLXQvmY7fb6woRMSD8cgk10UIJRt9nmuXl`,

## 핵심 결과

1. **UI 플로우 end-to-end 검증** — 브라우저로 폼 입력 → 27개 영상 수집(채널 10·플레이리스트 7·키워드 10) →
   "자막 생성" UI 버튼으로 transcript→Gemini POI→지오코딩까지 실행되는 2단계 게이팅 흐름을 확인했다.
2. **타임스탬프 버그(T-089) 수정 검증** — 직전 라이브 E2E에서 `varchar(16)` truncation으로 **실패하던
   키워드 소스가 이번엔 정상 완료**(`done`)했다. 컬럼을 `VARCHAR(64)`로 넓히고 `@validates` 클립을 둔
   수정이 실제로 동작함을 라이브로 확인했다.
3. **플레이리스트 full 파이프라인 성공** — 자막이 확보된 영상에서 9개 장소를 추출하고 **전부 지오코딩**했다:
   태종대, 감천문화마을, 광안리해수욕장, 국립해양박물관, 흰여울문화마을, 송도해상케이블카,
   이기대해안산책로, 국립일제강제동원역사관, 더베이101 (모두 부산, 좌표·카테고리 확보).

## 한계(이번 실행에서 관찰)

- **자막 가용성이 병목**: 27개 중 **3개 영상만** 유효한 `transcript_summary`를 얻었다. 나머지는 자막이
  비어 있어 POI 추출 단계에서 스킵됐다(채널·키워드 0건). 원인은 `youtube-transcript-api`가 오늘 다수의
  harvest로 누적 호출되어 rate-limit/차단된 것으로 보이며, `faster-whisper` 폴백은 opt-in
  (`TRANSCRIPT_WHISPER_ENABLED`)이라 활성화돼 있지 않았다. 영상 설명(`description_raw`)은 채워져 있으나,
  현재 파이프라인은 transcript가 비면 해당 영상을 건너뛰므로 #91(설명→POI)도 적용되지 않는다.
- 키워드 `제주도 가족여행`의 검색 결과는 shorts·음악·택시 광고 등 POI가 약한 영상이 많았다.

## 권장(별도 PR 제안)

1. **whisper 폴백 기본 활성화 또는 transcript 실패 시 description 단독 POI 경로**: transcript가 비어도
   영상 설명만으로 POI를 추출하도록 보완하면 자막 차단 환경에서도 결과가 나온다(#91 데이터는 이미 흐름).
2. `youtube-transcript-api` 차단 회피(프록시/쿠키) 또는 `yt-dlp` 자막 우선순위 조정.
